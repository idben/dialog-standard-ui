# 系統開發原則
1.清爽專業設計: 使用 https://shoelace.style/ 統一 UI 風格，遵循現代網頁設計原則，保持界面簡潔專業
2.技術使用: 使用簡單易用的開發語言，避免直接套用複雜的框架，使用多 agents 並行處理任務
3.復用既有資源: 優先使用既有文件和組件，避免重複建立相似功能，禁止製造一次性文件浪費資源與 token
4.階段性提交: 每個完整階段進行 commit 和 README 更新
5.完整測試驗證: 系統必須完整測試後才提供給用戶，確保功能正常運行
6.高效 Token 使用: 避免重複說明，專注核心功能實現和問題解決
7.全程使用正體中文進行互動，嚴格禁止使用簡體中文包含內容與用語文法。
8.合理使用 emoji，必免濫用。
9.主動分析後如果有需要拆解結構，==應詢問我== 是否根據功能拆分程式碼並取得同意。
10.客觀務實:用數據說話，減少主觀評價，語調客觀中性，不過度推銷，重點放在事實陳述，嚴禁說謊。
11.設計架構與功能必須完整符合 ISO27001:2022，並注意機敏資料不得上傳．

# 其他我的個人習慣
* 使用中文回覆
* 兩個空白，或一個 tab 等於兩空白的編排方式
* 如果有需要抓取元表素，不喜歡 getElement 系列語法，偏好 querySelector 系列語法
* 偏好小駝峰的變數命名方式
* 只要有做的修改，都先把步驟及細節寫入 task.md 中讓我看和與我討論完再執行。
* 從 task.md 執行完的內容，都需要在 task 中注記。
